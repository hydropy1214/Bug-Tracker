If your goal is to build a **professional web-based bug bounty platform** for **authorized security testing** (your own assets or programs that permit testing), then don't ask Replit AI to "build a scanner." Ask it to build a **modular enterprise application**.

Use the prompt below as your starting point.

# Build SentinelX - Enterprise Bug Bounty & Attack Surface Management Platform

You are a senior team consisting of:

* Principal Security Engineer
* Staff Backend Engineer
* Staff Frontend Engineer
* DevOps Engineer
* UI/UX Designer
* Database Architect
* QA Engineer

Your task is to build a production-grade web application named **SentinelX**.

The platform is ONLY intended for authorized security testing of assets owned by the user or explicitly covered by bug bounty or penetration testing authorization.

The project must be modular, scalable, secure, and suitable for deployment on a VPS using Docker.

## Development Plan

Implement in phases.

Phase 1:
Authentication, dashboard, projects, assets.

Phase 2:
Enumeration pipeline and scan orchestration.

Phase 3:
Security analysis plugin framework and reporting.

Phase 4:
AI assistant, notifications, scheduling, and performance tuning.

At the end of each completed phase:

* ensure the application builds successfully,
* all tests pass,
* Docker containers run correctly,
* database migrations are applied,
* APIs are documented,
* and the UI is fully functional before moving to the next phase.

This prompt will help Replit AI produce a solid application structure and workflow. For any actual security testing capabilities, keep them limited to targets you own or are explicitly authorized to assess, and integrate well-known tools responsibly rather than attempting to create an unrestricted offensive framework from scratch.

Once the skeleton is generated, you can iteratively expand the plugin system, reporting, dashboards, and integrations into a polished enterprise-grade platform.
