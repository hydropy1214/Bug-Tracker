### PART 1 – ACTIVE EXPLOITATION ENGINE (Phase 24)

The engine will execute the following modules in order. Each module:
- Receives the target URL, auth context (if any), and results from earlier phases.
- First sends a **baseline request** to ensure the endpoint is reachable and not rate‑limited.
- Uses **canary tokens** (unique strings like `SENTINELX_RCE_CANARY_<module>`) to confirm successful exploitation without causing damage.

#### Module 1: SQL Injection – Full Data Extraction
If Phase 12 detected error‑based or blind SQLi with high confidence:
- Determine the database type (MySQL, MSSQL, PostgreSQL) from error messages.

  - If the original query is on a table with user data, try to extract the first user row (email/hash). If you can, flag "Sensitive Data Leak – User Credentials".
- For time‑based blind:
  - Confirm with multiple time delays (5s, 3s) and if consistent, flag CRITICAL "Blind SQL Injection – RCE likely via stacked queries". Include recommendation to use sqlmap.

#### Module 2: NoSQL Injection – Authentication Bypass & Data Extraction
If Phase 12/22 found NoSQL operator injection (`$ne`, `$gt`, `$regex`) causing different responses:

- If the injection point is in a search or list endpoint, attempt to extract all documents using `{"$regex":".*"}`. If the response size increases significantly, flag "NoSQL Data Exposure".

#### Module 3: Command Injection – Real OS Command Execution
If Phase 12/22 flagged command injection:
- Send a safe payload: `; echo SENTINELX_CMDI_CANARY_$(whoami)` (Linux) or `| cmd /c echo SENTINELX_CMDI_CANARY_%USERNAME%` (Windows).

- For Twig/PHP: `{{_self.env.registerUndefinedFilterCallback('exec')}}{{_self.env.getFilter('echo SENTINELX_RCE')}}` – if canary appears, CRITICAL.

#### Module 5: SSRF – Cloud Metadata & Internal Network
If SSRF is suspected (parameter used to fetch URLs):
- Try to access cloud metadata endpoints:
  - AWS: `http://169.254.169.254/latest/meta-data/`
  - GCP: `http://metadata.google.internal/computeMetadata/v1/instance/?recursive=true`

#### Module 6: Path Traversal – File Read
