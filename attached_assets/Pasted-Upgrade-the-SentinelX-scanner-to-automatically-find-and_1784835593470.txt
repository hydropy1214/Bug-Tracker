## 5. ENHANCE SSTI DETECTION (in vuln-probes.ts)

In the SSTI module, apply these changes to eliminate false positives and confirm exploitation:

- **Payload reflection check**: before analyzing the response, verify that the literal injected payload (e.g., `{{7*7}}`) is actually present in the response body. If not, skip that parameter.
- **Unique canary**: use `{{'SENTINELX_SSTI_' + 'CONFIRM'}}` as the primary payload. If the exact string `SENTINELX_SSTI_CONFIRM` appears in the response, flag as VERIFIED with high confidence.
- **Multi-math verification**: if the math probe `{{7*7}}` yields "49", immediately send a second request with `{{8*8}}` and verify "64" appears. Only flag if both tests pass.
- **Proximity matching**: after confirming reflection, look for the result within 200 characters of the reflected payload.
- If the response status is 429 or 403 with WAF markers, skip that parameter entirely.
