## 3. Fix HTTP Request Smuggling false positives

In `phases/request-smuggling.ts`:
- Before running smuggling probes, send a baseline GET request.
- If `isWafOrRateLimit()` returns true, skip the smuggling phase.
- Also skip if the baseline response time > 500ms or status is not 2xx/3xx.
- For smuggling detection, do not flag a simple 400/501 response from a CDN as smuggling — require either a timing anomaly > 2 seconds AND a status change, or a confirmed desync (different response when re-requesting).
- Currently the scanner flags whenever status or time differs slightly — that's the cause of all false positives. Fix this.
